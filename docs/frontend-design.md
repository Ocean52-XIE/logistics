# 前端设计方案

## 1. 设计目标

前端需要同时满足两个方向：

- 对员工来说，学习路径清晰、使用门槛低、在短时间内找到该学内容。
- 对管理员来说，课程配置、培训跟踪、报表查看高效可控。

因此建议建设两个前端入口：

1. 员工学习端
2. 管理后台端

两端可以共享同一套技术栈与组件体系，但在导航结构和视觉层级上保持明显区分。

## 2. 技术选型

推荐采用以下方案：

- 框架：`Next.js 15 + React + TypeScript`
- 样式：`Tailwind CSS + CSS Variables`
- 组件：自建业务组件库，必要时引入 `Radix UI`
- 状态管理：`Zustand`
- 数据请求：`TanStack Query`
- 表单：`React Hook Form + Zod`
- 图表：`ECharts` 或 `Recharts`
- 视频播放：`video.js` 或浏览器原生播放器封装
- 国际化预留：`next-intl`

选择理由：

- `Next.js` 适合内容展示、管理后台和 SEO 友好的帮助页面统一建设。
- `TypeScript` 便于和后端共享接口类型约束。
- `TanStack Query` 可降低列表页、详情页、报表页的数据同步复杂度。

## 4. 信息架构

### 4.1 员工端信息架构

首页聚焦三个最关键问题：

- 我今天要学什么
- 我还差多少完成培训
- 我最近有哪些考试或通知

页面模块建议包括：

- 顶部欢迎区
- 今日待学卡片
- 学习进度总览
- 必修课程列表
- 推荐复习内容
- 最新通知
- 考试提醒
- 岗位认证状态

### 4.2 管理后台信息架构

后台导航建议按“内容、计划、执行、分析”四段组织：

- 内容：课程、章节、知识库、题库
- 计划：学习路径、培训计划、通知发布
- 执行：员工、组织、考试安排、完成情况
- 分析：报表、失分分析、组织对比、审计日志

这样管理员更容易建立完整心智模型。

## 5. 关键页面设计

## 5.1 员工端首页

目标：

- 让新员工进入首页后 10 秒内知道下一步操作。

关键模块：

- 学习任务横幅
- 本周培训进度条
- 必修课程卡片
- 待参加考试卡片
- 常用 SOP 快捷入口
- 通知公告流

交互要求：

- 首屏突出 1 个主要 CTA，例如“继续学习”。
- 已完成、进行中、逾期状态颜色区分明确。
- 重要通知需支持置顶和已读状态。

## 5.2 课程详情页

页面组成：

- 课程封面与简介
- 适用岗位、预计时长、完成规则
- 章节目录
- 学习进度
- 附件下载
- 讨论/答疑入口

关键交互：

- 视频、图文、PDF 章节统一学习体验。
- 支持章节完成打点。
- 支持“上一节 / 下一节”连续学习。
- 离开页面时自动保存学习进度。

## 5.3 在线考试页

页面组成：

- 考试说明
- 倒计时
- 答题区
- 题号导航
- 暂存状态
- 交卷确认

关键交互：

- 单选、多选、判断、案例题统一组件规范。
- 网络波动时自动暂存。
- 临近交卷前给出风险提醒。
- 交卷后展示分数、错题和建议复习内容。

## 5.4 知识库页面

页面组成：

- 分类筛选
- 搜索框
- 热门 SOP
- 最近更新
- 文章目录
- 关联课程

关键交互：

- 支持关键词高亮。
- 支持按岗位筛选内容。
- 支持从课程跳转到相关 SOP。

## 5.5 管理后台仪表盘

展示指标建议：

- 总员工数
- 在训员工数
- 本周完成率
- 首考通过率
- 逾期未完成人数
- 风险组织排行
- 热门课程排行
- 高频失分题目排行

仪表盘上方提供组织、岗位、时间范围筛选。

## 5.6 课程编辑页

建议采用“左侧目录 + 右侧编辑区”布局，支持以下能力：

- 创建章节
- 配置内容类型
- 拖拽排序
- 设置必修/选修
- 绑定题库
- 设置完成条件
- 预览发布

## 6. 角色体验设计

### 6.1 新员工体验原则

- 首页只暴露最少决策项，避免信息过载。
- 使用大卡片、强状态标签、清晰的进度条。
- 重点流程支持“继续上次学习”。

### 6.2 培训管理员体验原则

- 强调批量操作能力。
- 列表页支持高级筛选与导出。
- 重要配置提供预览和版本控制，防止误发布。

## 7. 视觉设计方向

不建议做成普通企业后台风格，而是做成“培训中枢 + 物流流程感”的界面语言。

### 7.1 视觉关键词

- 稳定
- 清晰
- 可信
- 具备流程感

### 7.2 推荐视觉方案

- 主色：深蓝 `#173B67`
- 强调色：橙色 `#F28C28`
- 辅助色：青绿 `#2D9C8C`
- 背景色：浅灰白 `#F5F7FA`
- 风险色：红色 `#D64545`

### 7.3 字体建议

- 中文：`HarmonyOS Sans SC` 或 `Source Han Sans SC`
- 数字与仪表盘：`IBM Plex Sans`

### 7.4 风格细节

- 首页可加入物流路径线条、节点图形、仓储格栅感背景。
- 关键卡片使用清晰阴影和层次，而不是堆叠过多边框。
- 仪表盘图表应偏“运营驾驶舱”风格，突出趋势与风险。

## 8. 组件设计

建议优先抽象以下组件：

- `ProgressCard`
- `CourseCard`
- `StatusBadge`
- `TrainingTaskBanner`
- `ExamQuestionPanel`
- `SopArticleViewer`
- `FilterBar`
- `StatMetricCard`
- `OrganizationSelector`
- `RoleTag`

统一规范：

- 状态色、间距、圆角、阴影通过 Design Tokens 管理。
- 所有表格与筛选区采用一致的交互模型。
- 表单错误提示位置固定，避免页面跳动。

## 9. 前端架构设计

推荐目录结构：

```txt
src/
  app/
  components/
  features/
    auth/
    courses/
    exams/
    knowledge-base/
    learning-paths/
    reports/
  services/
  store/
  hooks/
  schemas/
  styles/
  types/
```

模块划分建议：

- `features` 存放业务模块
- `services` 管理 API 请求封装
- `schemas` 存放 `zod` 校验
- `store` 存放轻量客户端状态

## 10. 权限控制

前端权限建议采用：

- 登录后获取用户角色与权限点
- 路由级鉴权
- 页面级功能按钮鉴权
- 数据级权限由后端决定，前端只做展示限制

常见权限点示例：

- `course:view`
- `course:edit`
- `exam:publish`
- `report:view_all`
- `user:assign_training`

## 11. 性能与可用性

- 课程列表、知识库列表支持懒加载与分页。
- 视频资源走 CDN。
- 图表和大型富文本模块按需加载。
- 长表单采用分段保存。
- 网络失败时给出明确提示和重试入口。

## 12. 响应式策略

虽然核心场景主要在电脑端，但建议支持平板和手机查看：

- 电脑端：完整学习和后台管理
- 平板端：学习、考试、主管查看报表
- 手机端：课程浏览、通知查看、轻量学习

不建议在首版手机端支持复杂后台编辑。

## 13. 可访问性与易用性

- 颜色对比度满足基础可读性要求。
- 表单和按钮提供明确聚焦态。
- 视频支持字幕和倍速。
- 关键培训流程支持键盘操作。

## 14. 埋点建议

为后续优化培训效果，前端需预留埋点：

- 首页任务点击率
- 课程开始率与完课率
- 章节停留时长
- 搜索关键词
- 考试中断率
- 交卷前返回修改行为
- 常见页面报错

这些数据可反向指导课程优化。
