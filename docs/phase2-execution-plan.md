# 第二阶段执行计划（运营增强）

## 1. 目标与周期

- 执行周期：2026-03-23 至 2026-05-01（6 周）
- 阶段目标：
  - 培训管理员不依赖线下表格即可完成日常培训运营。
  - 部门主管可识别风险员工并安排补训。
  - 平台补齐“组卷-考试-补考-提醒-分析-审计”运营闭环。

## 2. 范围与优先级

| 优先级 | 能力 | 说明 |
|---|---|---|
| P0 | 题库随机组卷 | 第二阶段核心地基，不完成则补考与失分分析无法闭环。 |
| P0 | 补考机制 | 建立未通过员工的可执行闭环。 |
| P0 | 通知中心 + 提醒任务 | 支撑开考、逾期、补考触达。 |
| P0 | 审计日志 | 覆盖关键管理动作，满足追溯要求。 |
| P0 | 组织/岗位筛选 + 失分分析（V1） | 支撑“风险识别”验收目标。 |
| P1 | 风险员工清单 + 一键补训 | 缩短“发现风险到处置”的路径。 |
| P2 | 高级筛选性能优化与通知策略精细化 | 在 P0/P1 稳定后迭代。 |

## 3. 里程碑（按周）

| 周次 | 里程碑 | 交付结果 |
|---|---|---|
| 第 1 周 | 数据与接口地基 | 完成题库、组卷规则、补考策略、审计日志、通知任务的数据结构与接口骨架；补齐 seed。 |
| 第 2-3 周 | 考试闭环 | 完成题库管理、随机组卷、发布考试、补考发起与状态流转。 |
| 第 4 周 | 触达闭环 | 完成通知中心与提醒任务（开考/逾期/补考），具备幂等和失败重试。 |
| 第 5 周 | 分析闭环 | 完成组织/岗位筛选与失分分析报表（V1），支持风险识别。 |
| 第 6 周 | 运营闭环与验收 | 完成风险员工清单与一键补训，回归测试、试运行、验收报告。 |

## 5. 验收与完成定义

### 阶段验收标准

- 培训管理员可在系统内完成：建题、组卷、发布、补考、通知、分析。
- 主管可在 10 分钟内识别风险员工并发起补训/补考。
- 报表支持组织/岗位筛选，且可查看失分分析。
- 审计日志覆盖关键动作并可查询追溯。

### 完成定义（DoD）

- 代码通过：`npm run typecheck`、`npm run build`、`npm test`。
- 核心流程 E2E 全绿，且关键接口具备幂等与错误重试验证。
- 试运行手册与验收清单同步更新到 `docs`。

## 6. 实施约束与边界

- 本阶段不引入复杂规则引擎，优先可运行的运营闭环。
- `admin` 后台权限策略保持不变，主管能力通过受限运营视图实现。
- 不在本阶段引入第三方外部系统深度集成（如 HR/OA 深度双向同步）。
